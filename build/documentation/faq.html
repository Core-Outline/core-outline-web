FAQ on using Core&Outline

Below you'll find answers to the questions we get asked the most.

You will find HTML pages in the public directory. If you want, you can include the file over there and make the project live. You can write SCSS which is generated to the CSS file in the public/assets directory. We also used a pug template engine to generate all HTML files which one is very helpful for easy customization.

For detailed documentation, check the following link: https://prium.github.io/Core&Outline/v3.17.0/documentation/getting-started.html.

Minimum requirements to run Core&Outline v3.17.0

  • Node v14.18.1
  • npm v6.14.5
  • Bootstrap v5.1.3

After running gulp you can start customizing the Core&Outline using Pug. You can start customizing from src/pug/pages/starter.pug where minimum styles and scripts are included to run Core&Outline theme.

Or you can update html files directly from the public directory.

You are not required to use pug, gulp, node, or any other workflow tools.All the HTML files are ready to use under the public folder. Just take a fresh copy of Core&Outline, open any HTML files in the public folder, and you can edit it as you like. Plane and simple HTML.

Updating from any earlier version of 3.x to v3.17.0:
Check out the following link and follow the changes: https://prium.github.io/Core&Outline/v3.17.0/changelog.html

Updating from version 2 to version 3:
As the latest versions of templates and all their packages have different structures behind their components, you have to update all the components manually into your project.
So, it would be better to start building applications with the Bootstrap 5 version of Core&Outline from scratch. You can include any components from Bootstrap 5 version to your project. Bring the component and its required SCSS and place it into your current project.

You will find the Figma link in the README.md file on your downloaded copy of Core&Outline. Here is the detailed documentation: https://prium.github.io/Core&Outline/v3.17.0/documentation/design-file.html.

If you still have any problem with the link, reach us out with your purchase ID at support@.com

Core&Outline is made out of HTML, CSS, and JS. You can not use Core&Outline as a "Native" component library for an android or iOS app. But if you use a web-view for your app or use a framework that uses HTML, CSS, and JS for mobile app UI, yes, you can use Core&Outline in that case

Many of our customers are using Core&Outline with Django. Using Core&Outline is like using Bootstrap with Django, just with some extra features.

For a starter, you need to take all the CSS and JS file from Core&Outline's public/assets folder to your project and link them to your view files. You can find all the required scripts and styles on the source of the starter page.

It is also possible to compile the SCSS and use the ES6 JavaScript partial files with Django using Core&Outline's built-in gulp based workflow.

We are afraid, there is no Blazor version of Core&Outline and no Blazor expert in our team at this moment. But we think, you would be able to use it just like Bootstrap, only an extended version.

In Core&Outline live demo, we are previewing both combo nav and top nav together. To apply top nav layout to any page, check the documentation in Core&Outline https://prium.github.io/Core&Outline/v3.17.0/modules/components/navs-and-tabs/top-navbar.html.

You will also get the example of navbar top layout page in public/demo/navbar-top.html. Also check the following link for top nav: https://prium.github.io/Core&Outline/v3.17.0/demo/navbar-top.html.

In the same way, you can implement the Combo nav layout. Here is the documentation of combo nav layout: https://prium.github.io/Core&Outline/v3.17.0/modules/components/navs-and-tabs/combo-navbar.html.

Example page:
https://prium.github.io/Core&Outline/v3.17.0/demo/combo-nav.html.

Why did we keep it different? Because in the demo, we included HTML codes for all the Navs on every page. But most of our users need to use only one nav and do not want their users to give the option to change the nav. That‚Äôs why we redirected the settings to the corresponding documentation page to see how they can configure it

Core&Outline works nicely with Laravel.

Good news is we have a starter project made with laravel mix and Core&Outline. This project is privately available. If you are interested, after purchasing Core&Outline, please send your purchase ID and we will send you the starter project.

Core&Outline works fine with Laravel 9x.

We have developed a very basic Angular starter pack to help developers get started with Core&Outline & Angular. The project is still a work in process, and we are updating it quite often to match with the industry best practices. If you are interested, after purchasing Core&Outline, please send your purchase ID and we will send you the starter pack.

Try deleting the package-lock.json and node_modules folder and then run npm i again.

I'm afraid we do not have any documentation on rails integration yet or any version for Ruby. But you may find the following link useful. Since all the Bootstrap themes are structured in a similar fashion, you may find it helpful: https://rubyyagi.com/how-to-integrate-html-bootstrap-theme-into-rails-6/.
